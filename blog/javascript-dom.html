What is JavaScript and its relationship to HTML and CSS?

Let’s pretend a website is a car. HTML forms the structure of the car and can be represented as the body and chassis defining where all elements of a car are positioned such as wheels, doors, windows, brakes, seats, steering wheel etc. Now let’s add CSS to the car. To do this we need to add JavaScript! Adding JavaScript adds parts to our car such as an engine, sensors and electrical components that allow us to operate and use the car for its intended purpose.

What is the DOM and how do we interact with it?

The DOM or Document Object Model can be viewed as a kind of tool that allows a browser to interact with the pages that make up the website.

Sometimes when we visit websites we want information to update/change as a response to different tasks we perform whilst navigating them. For example when we add an item to our cart we want that little number representing how many items are in our cart to increase by 1. For this to happen the website needs to be dynamic. The DOM is responsible for this dynamic response.

The DOM creates a version of the webpage that the browser can talk to and manipulate if it needs to using JavaScript, we call this the “document”. This document is divided into what we call “objects”.

Each object represents all the different elements on our page such as heading, paragraphs and buttons (or our cart). The DOM has tools that can change and manipulate objects. In response to triggers from the user such as button clicks the DOM can target and update objects by changing their properties or cause action by calling methods. Using the DOM we are able to create exciting, dynamic websites that customize our user experience.

What's the difference between "arrays" and "objects"?

Both arrays and objects can be used for storing and retrieving data. However, the key difference is HOW they store and retrieve this data.

Arrays store an ordered collection of items/data. Say we have a box of 5 spanners. The array in this case is represented as the box, and the spanners are the collection of items the array holds. Each spanner has a different length and has its own slot within the box numbered 0-4 (slot 1 = 0, slot 2 = 1, … slot 5 = 4). It is important to note that an array organizes the items it contains by index values starting from 0. The slots within the box represent the index values of items stored in an array, each item having its own unique index value. If I was to ask someone to retrieve a spanner for me I would have to tell them the number of the slot that the spanner is in for them to be able to retrieve the right one. The same goes for arrays. Functions - What are they and why are they helpful?

Functions are like little reuseable packages of code that perform some sort of procedure. Essentially there’s two parts to a function: Defining it and calling it.

First we need to define the function. Say there’s a task we want to complete over and over and requires writing numerous lines of code every time we want to action it. Instead of typing out each line of code every time we want to complete the task we can assign the block of code to a function name. The function name now represents our wee (or large) block of code.

Now we can call the function. Everytime we want to action the task we defined in our function, instead of typing it out again, we just use the function name. By doing this we save time and it also makes our JavaScript code more readable, organized and reusable. Another great benefit of this is if we want to change any part of the task being executed we only have to make changes once as we only need to update the section where we defined the function.
